# Customer Requirement Document (v0.1)
## 클라우드 파일 공유 시스템 (미니 드라이브)

---

## 1. Background

우리 조직은 여러 팀이 동시에 협업하는 프로젝트 기반 환경에서 일하고 있다.  
현재 팀원들은 문서, 설계 파일, 보고서, 이미지 및 기타 프로젝트 자료를 이메일, 메신저, 개인 클라우드 서비스 등을 통해 공유하고 있다.

이러한 방식은 다음과 같은 문제를 발생시킨다:

- 파일의 최신 버전 관리가 어렵다  
- 파일을 찾는 데 시간이 많이 소요된다  
- 외부 협력 업체와의 파일 공유 시 보안 문제가 발생할 수 있다  

이러한 문제를 해결하기 위해 조직 내부에서 사용할 수 있는 클라우드 기반 파일 관리 및 협업 시스템을 구축하고자 한다.  
해당 시스템은 파일 저장, 관리, 공유 기능을 제공해야 한다.

---

## 2. Overall Goals

시스템은 다음 목표를 달성해야 한다:

- 조직 내 파일 중앙 관리
- 파일 업로드 및 다운로드 지원
- 사용자 간 파일 공유 기능 제공
- 파일 검색 기능 제공
- 외부 사용자와의 안전한 파일 공유

> 선택사항: 웹 브라우저 기반 접근 (별도 프로그램 설치 없이 사용)

---

## 3. Functional Requirements

### 3.1 File Storage

- 사용자는 파일을 업로드하여 저장할 수 있어야 한다
- 다양한 파일 형식 지원 (문서, 이미지, PDF, 압축, 영상 등)
- 파일 업로드는 직관적이고 간단해야 한다
- 대용량 파일 업로드 지원
- 업로드된 파일 다운로드 가능

파일 메타데이터 저장:
- 파일 이름
- 업로드 날짜
- 파일 크기

---

### 3.2 Folder Organization

- 폴더 생성 / 수정 / 삭제 가능
- 다단계 폴더 구조 지원
- 파일 이동 기능 제공

추가 기능:
- 삭제된 파일 일정 기간 복구 기능

---

### 3.3 File Sharing

- 사용자 간 파일 공유 가능
- 공유 권한 설정 가능:
  - 읽기
  - 수정
  - 댓글

외부 공유:
- 링크 기반 파일 공유
- 링크 만료 기간 설정 기능

---

### 3.4 File Search

- 파일 이름 기반 검색
- 검색 필터 기능:
  - 날짜 기준
  - 사용자 기준
  - 파일 유형 기준

- 빠른 검색 결과 제공

---

### 3.5 File Version Management

- 파일 버전 관리 기능 제공
- 이전 버전 목록 확인
- 이전 버전 다운로드
- 이전 버전 복원

> 모든 버전을 무제한 저장할 필요는 없음

---

### 3.6 User Accounts

- 이메일 + 비밀번호 기반 로그인

관리자 기능:
- 사용자 계정 생성
- 사용자 계정 삭제
- 사용자 저장 공간 관리

선택 기능:
- 사용자 그룹 관리

---

## 7. Quality Attributes (프로젝트 II)

본 시스템의 품질 요소는 수업 자료에서 제시한 스테이크홀더 관점과 소프트웨어 품질 모델을 바탕으로 도출하였다. 
Mini Drive는 클라우드 기반 파일 공유 시스템이므로, 사용자가 체감하는 외적 품질과 개발 및 운영 조직이 중요하게 보는 내적 품질을 함께 고려해야 한다.

### 7.1 사용자 관점의 기대치

사용자는 파일 업로드/다운로드, 폴더 관리, 파일 검색, 파일 공유 및 버전 관리 기능이 정확하게 동작하기를 기대한다. 
따라서 기능 정확성이 중요하다. 또한 파일 유실 없이 안정적으로 저장되고 복구될 수 있어야 하므로 신뢰성과 복구 가능성이 중요하다. 
사용자는 별도의 교육 없이도 쉽게 사용할 수 있는 인터페이스를 원하므로 사용성 또한 핵심 품질 요소이다. 
파일 검색 결과는 빠르게 제공되어야 하고, 대용량 파일 업로드/다운로드도 효율적으로 처리되어야 하므로 성능 효율성이 필요하다. 
외부 링크 공유 기능이 존재하므로 권한 없는 접근과 데이터 유출을 방지하기 위한 보안성도 중요한 품질 요소이다.

### 7.2 개발자 및 유지보수 관점의 기대치

개발자와 유지보수 담당자는 파일 관리, 공유, 검색, 계정 관리, 버전 관리 기능이 모듈화되어 유지보수가 쉬운 구조를 원한다. 
따라서 유지보수성이 중요하다. 또한 권한 설정, 버전 복원, 검색 필터 등의 기능을 자동화 테스트로 검증하기 쉬워야 하므로 시험 가능성이 중요하다. 
소스 코드의 가독성과 표준화, 요구사항·설계·구현 간의 일관된 문서화도 필요하다. 
나아가 다양한 브라우저와 배포 환경에서 쉽게 적용될 수 있도록 이식성과 적응성도 고려해야 한다.

### 7.3 관리자 관점의 기대치

관리자는 제한된 비용과 일정 안에서 안정적인 시스템이 개발되기를 기대한다. 
따라서 비용 효율성과 프로젝트 관리 가능성이 중요하다. 
동시 사용자 약 200명을 안정적으로 지원하고 서비스 중단을 최소화해야 하므로 가용성과 운영 안정성이 필요하다. 
또한 조직 내부 시스템으로서 데이터 보호와 접근 통제가 보장되어야 하므로 보안성이 매우 중요하다. 
향후 공동 편집, 자동 분류, AI 기반 검색 기능으로 확장될 가능성을 고려하면 확장성도 중요한 품질 요소이다. 
마지막으로 테스트 결과, 보안 취약점, 요구사항 반영 여부 등을 관리자가 확인할 수 있도록 품질 가시성도 확보되어야 한다.
