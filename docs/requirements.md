# Customer Requirement Document (v0.1)
## 클라우드 파일 공유 시스템 (미니 드라이브)

---

## 1. Background

우리 조직은 여러 팀이 동시에 협업하는 프로젝트 기반 환경에서 일하고 있다.  
현재 팀원들은 문서, 설계 파일, 보고서, 이미지 및 기타 프로젝트 자료를 이메일, 메신저, 개인 클라우드 서비스 등을 통해 공유하고 있다.

이러한 방식은 다음과 같은 문제를 발생시킨다:

- 파일의 최신 버전 관리가 어렵다  
- 파일을 찾는 데 시간이 많이 소요된다  
- 외부 협력 업체와의 파일 공유 시 보안 문제가 발생할 수 있다  

이러한 문제를 해결하기 위해 조직 내부에서 사용할 수 있는 클라우드 기반 파일 관리 및 협업 시스템을 구축하고자 한다.  
해당 시스템은 파일 저장, 관리, 공유 기능을 제공해야 한다.

---

## 2. Overall Goals

시스템은 다음 목표를 달성해야 한다:

- 조직 내 파일 중앙 관리
- 파일 업로드 및 다운로드 지원
- 사용자 간 파일 공유 기능 제공
- 파일 검색 기능 제공
- 외부 사용자와의 안전한 파일 공유

> 선택사항: 웹 브라우저 기반 접근 (별도 프로그램 설치 없이 사용)

---

## 3. Functional Requirements

### 3.1 File Storage

- 사용자는 파일을 업로드하여 저장할 수 있어야 한다
- 다양한 파일 형식 지원 (문서, 이미지, PDF, 압축, 영상 등)
- 파일 업로드는 직관적이고 간단해야 한다
- 대용량 파일 업로드 지원
- 업로드된 파일 다운로드 가능

파일 메타데이터 저장:
- 파일 이름
- 업로드 날짜
- 파일 크기

---

### 3.2 Folder Organization

- 폴더 생성 / 수정 / 삭제 가능
- 다단계 폴더 구조 지원
- 파일 이동 기능 제공

추가 기능:
- 삭제된 파일 일정 기간 복구 기능

---

### 3.3 File Sharing

- 사용자 간 파일 공유 가능
- 공유 권한 설정 가능:
  - 읽기
  - 수정
  - 댓글

외부 공유:
- 링크 기반 파일 공유
- 링크 만료 기간 설정 기능

---

### 3.4 File Search

- 파일 이름 기반 검색
- 검색 필터 기능:
  - 날짜 기준
  - 사용자 기준
  - 파일 유형 기준

- 빠른 검색 결과 제공

---

### 3.5 File Version Management

- 파일 버전 관리 기능 제공
- 이전 버전 목록 확인
- 이전 버전 다운로드
- 이전 버전 복원

> 모든 버전을 무제한 저장할 필요는 없음

---

### 3.6 User Accounts

- 이메일 + 비밀번호 기반 로그인

관리자 기능:
- 사용자 계정 생성
- 사용자 계정 삭제
- 사용자 저장 공간 관리

선택 기능:
- 사용자 그룹 관리

---

## 4. Non-functional Requirements

### 4.1 Security

- 로그인 인증
- 권한 기반 접근 제어
- 데이터 전송 암호화
- 외부 공유 시 보안 유지

---

### 4.2 Performance

- 약 200명 동시 사용자 지원
- 안정적인 시스템 운영

성능 요구:
- 파일 검색 빠른 응답
- 파일 다운로드 빠른 처리

---

## 5. Constraints

- 웹 기반 시스템 (권장)

## 7. Quality Attributes (프로젝트 II)

수업에서 제시한 스테이크홀더 관점에 따라  
Mini Drive 시스템의 품질 기대치를 다음과 같이 도출한다.

---

### 7.1 사용자가 원하는 기대치는 무엇인가?

- 사용자는 파일 업로드/다운로드, 공유, 권한 설정 기능이 정확하게 동작하기를 원함
- 사용자는 파일이 유실되지 않고 안정적으로 저장되기를 원함
- 사용자는 별도의 학습 없이 쉽게 사용할 수 있는 시스템을 원함
- 사용자는 파일 검색 결과가 빠르게 제공되기를 원함
- 사용자는 대용량 파일도 지연 없이 업로드 및 다운로드되기를 원함
- 사용자는 외부 공유 시에도 데이터가 안전하게 보호되기를 원함
- 사용자는 오류 상황에서도 시스템이 중단되지 않고 정상적으로 동작하기를 원함

---

### 7.2 개발자 및 유지보수 담당자가 원하는 기대치는 무엇인가?

- 개발자는 시스템이 모듈화되어 유지보수와 확장이 용이하기를 원함
- 개발자는 기능이 독립적으로 테스트 가능한 구조를 원함
- 개발자는 코드가 읽기 쉽고 이해하기 쉬운 구조로 작성되기를 원함
- 개발자는 요구사항과 코드 간의 관계를 추적할 수 있기를 원함
- 개발자는 공통 기능이 재사용 가능한 구조로 설계되기를 원함
- 개발자는 다양한 환경에서도 쉽게 실행 가능한 시스템을 원함

---

### 7.3 개발 조직의 관리자가 원하는 기대치는 무엇인가?

- 관리자는 제한된 비용과 일정 내에서 시스템이 개발되기를 원함
- 관리자는 시스템이 안정적으로 운영되고 중단 없이 서비스되기를 원함
- 관리자는 사용자 증가에도 문제없이 확장 가능한 시스템을 원함
- 관리자는 조직 내부 데이터가 안전하게 보호되기를 원함
- 관리자는 시스템의 상태와 품질을 쉽게 확인할 수 있기를 원함
