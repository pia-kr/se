## 대상 시스템 품질 요소 측정하기 (프로젝트 II)

1장의 프로젝트 정의서에서 제시한 핵심 기능과 유사 시스템 분석 결과를 기반으로, 대상 시스템에서 중요하게 고려해야할 품질 요소를 추정하고 정리

---

### 1 사용자가 원하는 기대치는 무엇인가?

- 사용자는 파일 업로드/다운로드, 공유, 권한 설정 기능이 정확하게 동작하기를 원함
- 사용자는 파일이 유실되지 않고 안정적으로 저장되기를 원함
- 사용자는 별도의 학습 없이 쉽게 사용할 수 있는 시스템을 원함
- 사용자는 파일 검색 결과가 빠르게 제공되기를 원함
- 사용자는 대용량 파일도 지연 없이 업로드 및 다운로드되기를 원함
- 사용자는 외부 공유 시에도 데이터가 안전하게 보호되기를 원함
- 사용자는 오류 상황에서도 시스템이 중단되지 않고 정상적으로 동작하기를 원함

---

### 2 개발자 및 유지보수 담당자가 원하는 기대치는 무엇인가?

- 개발자는 시스템이 모듈화되어 유지보수와 확장이 용이하기를 원함
- 개발자는 기능이 독립적으로 테스트 가능한 구조를 원함
- 개발자는 코드가 읽기 쉽고 이해하기 쉬운 구조로 작성되기를 원함
- 개발자는 요구사항과 코드 간의 관계를 추적할 수 있기를 원함
- 개발자는 공통 기능이 재사용 가능한 구조로 설계되기를 원함
- 개발자는 다양한 환경에서도 쉽게 실행 가능한 시스템을 원함

---

### 3 개발 조직의 관리자가 원하는 기대치는 무엇인가?

- 관리자는 제한된 비용과 일정 내에서 시스템이 개발되기를 원함
- 관리자는 시스템이 안정적으로 운영되고 중단 없이 서비스되기를 원함
- 관리자는 사용자 증가에도 문제없이 확장 가능한 시스템을 원함
- 관리자는 조직 내부 데이터가 안전하게 보호되기를 원함
- 관리자는 시스템의 상태와 품질을 쉽게 확인할 수 있기를 원함
