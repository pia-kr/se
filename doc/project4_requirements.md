# 1. 서론

## 1.1 문서의 목적 및 범위

본 문서는 **Mini Drive(클라우드 파일 공유 시스템)** 개발에 필요한 요구사항을 정의하기 위한 문서이다.  

시스템이 제공해야 하는 기능적 요구사항과 비기능적 요구사항, 인터페이스 요구사항을 명확히 하여 개발 과정의 기준으로 활용한다.

---

## 1.2 대상 시스템 개요

Mini Drive는 조직 내부 파일을 중앙에서 관리하고 사용자 간 공유를 지원하는 웹 기반 클라우드 시스템이다.

### 주요 기능

| ID | 설명 |
|----|------|
| F01 | 파일 업로드 및 다운로드 |
| F02 | 폴더 생성 / 삭제 / 이동 |
| F03 | 사용자 간 파일 공유 |
| F04 | 권한 설정 (읽기/수정/댓글) |
| F05 | 외부 링크 공유 |
| F06 | 파일 검색 및 필터링 |
| F07 | 파일 버전 관리 |
| F08 | 관리자 계정 관리 |

---

## 1.3 용어 정의

| 용어 | 설명 |
|------|------|
| 클라우드 저장소 | 인터넷 기반 파일 저장 공간 |
| 버전 관리 | 파일 변경 이력을 저장하고 복구하는 기능 |
| 외부 공유 링크 | 외부 사용자에게 제공되는 접근 URL |
| 메타데이터 | 파일 이름, 크기, 업로드 시간 등의 정보 |
