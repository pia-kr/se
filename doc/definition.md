# 프로젝트 정의서

---

## 1. 프로젝트 명칭

클라우드 파일 공유 시스템

---

## 2. 개발 시스템 명칭

Mini Drive (미니 드라이브)

---

## 3. 개발 시스템 설명

본 시스템은 조직 내에서 파일을 효율적으로 저장, 관리, 공유하기 위한 클라우드 기반 파일 관리 시스템이다.  
사용자는 웹 브라우저를 통해 파일을 업로드하고 다운로드할 수 있으며, 폴더 구조를 활용하여 파일을 체계적으로 관리할 수 있다.  
또한 사용자 간 파일 공유 및 외부 링크 공유 기능을 제공하여 협업을 지원한다.  
보안 기능과 검색 기능을 통해 안정적이고 편리한 파일 관리 환경을 제공하는 것을 목표로 한다.

---

## 4. 핵심 기능

- 파일 업로드 및 다운로드 기능
- 폴더 생성, 수정, 삭제 및 계층 구조 관리
- 사용자 간 파일 공유 및 권한 설정 기능
- 링크 기반 외부 파일 공유 기능 (만료 기간 설정 포함)
- 파일 검색 기능 (이름, 날짜, 사용자, 유형 기준)
- 파일 버전 관리 기능 (이전 버전 조회 및 복원)
- 사용자 계정 관리 기능 (로그인, 관리자 기능 포함)

---

## 5. 기존 유사 소프트웨어 조사 및 분석

### 1) Google Drive

- 장점:
  - 강력한 파일 공유 및 협업 기능
  - 실시간 문서 공동 편집 지원
  - 다양한 파일 형식 지원

- 단점:
  - 무료 용량 제한
  - 조직 맞춤형 커스터마이징 어려움

- 프로젝트 반영:
  - 협업 중심의 파일 공유 및 권한 설정 기능을 적극 반영
  - 조직 환경에 맞는 **맞춤형 권한 관리 기능**을 강화하여 단점 보완

---

### 2) Dropbox

- 장점:
  - 간단하고 직관적인 UI
  - 안정적인 파일 동기화 기능
  - 다양한 플랫폼 지원

- 단점:
  - 무료 사용자 기능 제한
  - 협업 기능이 상대적으로 제한적

- 프로젝트 반영:
  - 직관적인 UI 구조를 참고하여 **사용 편의성 중심 설계**
  - 불필요한 기능을 줄이고 핵심 기능 중심으로 구성

---

### 3) OneDrive

- 장점:
  - Microsoft 서비스와 높은 연동성
  - 기업 환경에 적합
  - 문서 관리 기능 우수

- 단점:
  - UI가 다소 복잡함
  - 타 플랫폼 사용자 접근성 제한

- 프로젝트 반영:
  - 기업 환경에 적합한 **파일 관리 및 권한 체계 설계 참고**
  - 복잡한 UI를 단순화하여 접근성 개선

---

### ▶ 종합 분석

기존 시스템들은 강력한 기능을 제공하지만,  
복잡하거나 조직 맞춤화에 한계가 있는 경우가 많다.

따라서 본 프로젝트는 다음과 같은 방향으로 설계한다:

- 단순하고 직관적인 사용자 인터페이스 제공 (Dropbox 참고)
- 조직 맞춤형 권한 관리 기능 강화 (Google Drive 단점 보완)
- 기업 환경에 적합한 파일 관리 구조 설계 (OneDrive 참고)
- 불필요한 기능을 제거한 경량화된 시스템 구현

결론적으로, 기존 시스템의 장점은 유지하고 단점은 보완한  
**조직 최적화형 파일 공유 시스템**을 목표로 한다.
