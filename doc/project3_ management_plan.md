# 1. 서론

## 1.1 문서의 목적 및 범위

본 문서는 **Mini Drive 프로젝트**의 체계적인 수행과 효율적인 관리를 위해 작성되었다.

### 목적

- 프로젝트 목표 및 범위 명확화
- 일정 및 역할 분담 정의
- 개발 절차 및 품질 관리 기준 수립
- 팀원 간 원활한 의사소통 지원
- 성공적인 프로젝트 완료 지원

### 범위

Mini Drive는 조직 내부에서 사용하는 **웹 기반 클라우드 파일 공유 시스템**으로 다음 기능을 포함한다.

- 파일 업로드 / 다운로드
- 폴더 구조 관리
- 사용자 간 공유 및 권한 설정
- 외부 링크 공유
- 파일 검색
- 버전 관리
- 관리자 기능

---

## 1.2 프로젝트 개요

기존에는 이메일, 메신저, 개인 클라우드 등 여러 경로로 파일을 관리하여 다음과 같은 문제가 발생하였다.

- 최신 파일 버전 확인 어려움
- 자료 검색 시간 증가
- 공유 권한 통제 어려움
- 외부 공유 시 보안 문제 발생

Mini Drive는 이를 해결하기 위해 중앙 집중형 파일 관리 시스템을 구축한다.

# 4. 기술 관리 계획

## 4.1 버전 관리

GitHub를 활용하여 버전을 관리한다.

### 브랜치 전략

- `main` : 최종 안정 버전
- `develop` : 통합 개발 버전
- `feature/*` : 기능 개발 브랜치

### 커밋 규칙

- `feat:` 기능 추가
- `fix:` 오류 수정
- `docs:` 문서 수정
- `refactor:` 코드 개선
- `test:` 테스트 추가

---

## 4.2 형상 관리

관리 대상은 다음과 같다.

- 소스 코드

- 테스트 결과

변경 사항은 Pull Request 후 병합한다.
