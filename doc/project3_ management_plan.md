# 1. 개요 (Introduction)

## 1.1 문서의 목적

본 문서는 **Mini Drive(클라우드 파일 공유 시스템)** 개발을 위한 프로젝트 관리 기준을 정의한다.  

프로젝트의 일정, 역할 분담, 개발 절차, 품질 목표, 산출물 관리 방법 등을 명확히 하여 프로젝트를 체계적으로 수행하는 것을 목적으로 한다.

---

## 1.2 프로젝트 개요

Mini Drive는 조직 내부에서 사용하는 경량형 클라우드 파일 공유 시스템이다.

기존 이메일, 메신저, 개인 클라우드에 분산되어 있던 파일을 중앙에서 관리하며 다음 기능을 제공한다.

- 파일 업로드 / 다운로드
- 폴더 구조 관리
- 사용자 간 공유 및 권한 설정
- 외부 링크 공유
- 파일 검색
- 버전 관리
- 관리자 계정 관리

---

## 1.3 관련 문서

- 프로젝트 정의서  
- 고객 요구사항 문서(Customer Requirement Document)  
- 품질 요소 분석 문서

# 2. 개발 계획 (Development Plan)

## 2.1 프로젝트 모델 및 개발 방법론

본 프로젝트는 요구사항 변경 가능성이 있으므로 **Agile + Scrum 방식**을 적용한다.

운영 방식:

- 1~2주 단위 Sprint 진행
- 기능 우선순위 기반 개발
- 반복적 개선
- 정기 회의 및 피드백 반영

---

## 2.2 작업 분할도 (WBS)

### 1단계: 기획

- 프로젝트 정의
- 요구사항 수집
- 유사 시스템 조사

### 2단계: 설계

- DB 설계
- 시스템 구조 설계
- UI 화면 설계

### 3단계: 구현

- 로그인 기능
- 파일 업로드/다운로드
- 공유 기능
- 검색 기능
- 관리자 기능

### 4단계: 테스트

- 기능 테스트
- 통합 테스트
- 오류 수정

### 5단계: 배포

- GitHub 배포
- 사용자 시연
- 최종 발표

---

## 2.3 소요 자원 (Staffing, Cost)

| 구분 | 내용 |
|------|------|

# 4. 기술 관리 계획 (Technical Management)

## 4.1 버전 관리

GitHub를 사용하여 소스코드 버전을 관리한다.

규칙:

- main : 배포 안정 버전
- develop : 통합 개발 버전
- feature/* : 기능 개발 브랜치

Commit 예시:

- feat: 로그인 기능 추가
- fix: 업로드 오류 수정
- docs: README 수정

---

## 4.2 형상 관리

관리 대상:

- 소스코드
- 문서
- DB 스키마
- 발표 자료
- 테스트 결과

변경 시 Pull Request 기반 리뷰 후 반영한다.

---

## 4.3 기술 관리

사용 기술 스택:

- Frontend: React / HTML CSS JS
- Backend: Spring Boot
- Database: MySQL
- Deploy: GitHub / Render / AWS (선택)
